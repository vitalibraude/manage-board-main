# 🚀 הוראות הפעלת GitHub Pages - צעד אחר צעד

## ✅ שלב 1: הכנס ל-Repository שלך ב-GitHub

פתח את הדפדפן וכנס לכתובת:
```
https://github.com/vitalibraude/manage-board-main
```

---

## ⚙️ שלב 2: כנס להגדרות (Settings)

1. בדף ה-Repository, חפש את כפתור **"Settings"** (⚙️) בתפריט העליון
2. לחץ עליו

![Settings Location](https://docs.github.com/assets/cb-28266/mw-1440/images/help/repository/repo-actions-settings.webp)

---

## 📄 שלב 3: מצא את סעיף Pages

1. בתפריט הצד **השמאלי**, גלול למטה
2. חפש את הסעיף **"Pages"** (תחת קטגוריית "Code and automation")
3. לחץ עליו

---

## 🔧 שלב 4: הגדר את המקור (Source)

בסעיף **"Build and deployment"**:

1. תחת **"Source"**, בחר באפשרות: **"Deploy from a branch"**

2. תחת **"Branch"**:
   - בחר את הענף: **`main`**
   - בחר את התיקייה: **`/ (root)`**

3. לחץ על כפתור **"Save"** (שמור)

### תצוגה חזותית של ההגדרות:
```
Source: [Deploy from a branch]

Branch: [main ▼]  [/ (root) ▼]  [Save]
```

---

## ⏳ שלב 5: המתן לפריסה (Deployment)

1. לאחר השמירה, GitHub יתחיל לבנות ולפרוס את האתר
2. זה לוקח בדרך כלל **1-3 דקות**
3. תראה הודעה ירוקה שמציגה את כתובת האתר

---

## 🎉 שלב 6: גש לאתר שלך!

כשהפריסה הושלמה, האתר שלך יהיה זמין בכתובת:

```
https://vitalibraude.github.io/manage-board-main/
```

---

## 🔄 עדכון האתר

**שים לב:** כל עדכון ש-push לענף `main` יגרום לעדכון אוטומטי של האתר!

```bash
# לדוגמה, אם תרצה לעדכן משהו:
git add .
git commit -m "Update task board"
git push origin main

# GitHub Pages יתעדכן אוטומטית תוך כמה דקות
```

---

## ❓ פתרון בעיות נפוצות

### האתר לא נטען?
1. וודא שבחרת את הענף `main` ולא ענף אחר
2. וודא שבחרת `/ (root)` ולא תיקייה אחרת
3. המתן 5-10 דקות נוספות - לפעמים זה לוקח זמן

### שגיאת 404?
1. וודא שקיים קובץ `index.html` בתיקייה הראשית
2. נסה לנקות את ה-cache של הדפדפן (Ctrl+Shift+R / Cmd+Shift+R)

### האתר לא מתעדכן?
1. כנס ל-"Actions" בתפריט העליון של ה-Repository
2. וודא שה-workflow האחרון הצליח (סימן ✅ ירוק)
3. אם יש שגיאה, לחץ על ה-workflow לפרטים נוספים
